Step 1 of 7

Create your Telegram bot

  1. Open Telegram and search for @BotFather (the account with the blue checkmark).
  2. Open the chat, press Start, then send /newbot.
  3. Choose a name, then a username that ends in bot.
  4. BotFather replies with a line "Use this token to access the HTTP API:" — copy that token and paste it below.
Step 2 of 7

Your Telegram username

  1. In Telegram, open Settings (desktop: ☰ menu → Settings; phone: Settings tab, bottom-right).
  2. Tap Edit profile → Username and set one if you don't have it yet.
  3. Type it below without the leading @. Only this user may control the bot.
Step 3 of 7

Where are your projects?

  1. Single folder: point Crundi at one parent folder — every subfolder becomes a project.
  2. Multiple folders: add projects individually (with full paths) later from the web UI.
  3. For single folder, press Browse and pick the folder that holds your code.
Projects will be added individually with their full paths from the web UI.
Step 4 of 7 · optional

Remote access (optional)

  1. Skip this to use a free random quick tunnel — Crundi generates a URL each start.
  2. For a fixed custom domain: open Cloudflare Zero Trust → Networks → Tunnels → Create a tunnel → Cloudflared.
  3. Name it. On the "Install and run a connector" screen, only copy the token from the install command (the long eyJ... string) and paste it below.
  4. Add a Public Hostname for the tunnel and enter that full URL below.
Don't install Cloudflare's connector. Cloudflare's page tells you to download/run cloudflared (or a .msi/service) to "install the connector" — skip that entirely. Crundi already bundles and runs cloudflared itself; installing it again creates a second connector that fights ours and breaks the tunnel. You only need the token and the public hostname URL from that screen.

Leave empty (or press Skip) to use a random quick tunnel.

Step 5 of 7

Enable Telegram login

The web UI logs in with the "Log in with Telegram" button. Telegram only shows it on a domain you've registered for your bot.

  1. Press Save & Start below. When Crundi is running it shows your public URL (your custom domain, or the random quick-tunnel URL in the logs).
  2. Open @BotFather → send /setdomain (or /mybots → your bot → Bot Settings → Domain).
  3. Select your bot and send that full URL (e.g. https://crundi.yourdomain.com) as the domain.
Use only the bare domain/URL — nothing else. Don't add a path, port, or trailing text, and don't enter http://localhost; Telegram rejects those and the login button won't appear. If you use a random quick tunnel, the URL changes every launch, so you'd have to re-run /setdomain each time — a fixed custom domain (previous step) is strongly recommended for login.
