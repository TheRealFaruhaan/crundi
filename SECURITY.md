# Security Policy

Crundi gives a single authenticated user full control of the host it runs on —
an interactive shell, file read/write, service control, and screen capture. Run
it only for yourself, keep your tunnel URL private, set your bot's login domain,
and never share your bot token.

## Reporting a vulnerability

Please do **not** open a public issue for security problems. Use GitHub's
**private vulnerability reporting**: the repository's **Security** tab →
**"Report a vulnerability"**. Reports stay private to the maintainer.
